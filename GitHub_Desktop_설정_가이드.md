# GitHub Desktop 사용 가이드 (토큰 불필요!)

## 🎯 GitHub Desktop이란?

- GitHub 공식 GUI 프로그램
- 토큰 불필요 (자동 인증)
- 드래그 앤 드롭으로 쉽게 사용
- 충돌 해결 쉬움

## 📥 설치 방법

### 1단계: 다운로드
1. 브라우저에서 https://desktop.github.com/ 접속
2. **"Download for Windows"** 클릭
3. 설치 파일 다운로드

### 2단계: 설치
1. 다운로드한 파일 실행
2. 기본 설정으로 설치 진행
3. 설치 완료 후 GitHub Desktop 실행

### 3단계: GitHub 계정 로그인
1. GitHub Desktop 실행
2. **"Sign in to GitHub.com"** 클릭
3. 브라우저가 열리면 GitHub 계정으로 로그인
4. **"Authorize GitHub Desktop"** 클릭
5. 자동으로 인증 완료!

## 🔗 기존 프로젝트 연결

### 방법 1: 로컬 저장소 추가
1. GitHub Desktop 실행
2. **"File"** → **"Add Local Repository"** 클릭
3. **"Choose..."** 클릭
4. 프로젝트 폴더 선택: `C:\Users\park.ji.hun\Desktop\CODE\HANA`
5. **"Add Repository"** 클릭

### 방법 2: GitHub에서 클론
1. GitHub Desktop 실행
2. **"File"** → **"Clone Repository"** 클릭
3. **"GitHub.com"** 탭 선택
4. `jadu329-cmd/Hana_2026` 찾기
5. **"Clone"** 클릭

## 📤 사용 방법

### 파일 수정 후 저장
1. Cursor에서 파일 수정
2. 파일 저장 (Ctrl + S)
3. GitHub Desktop 자동으로 변경사항 감지

### 커밋 및 푸시
1. GitHub Desktop 왼쪽에서 변경된 파일 확인
2. 하단에 커밋 메시지 입력 (예: "심방리스트 수정")
3. **"Commit to main"** 클릭
4. **"Push origin"** 클릭 (또는 상단 메뉴에서)
5. 완료! 토큰 입력 불필요!

## 🔄 다른 컴퓨터에서 사용

### 첫 번째 컴퓨터
1. GitHub Desktop으로 파일 수정
2. Commit → Push
3. 자동으로 GitHub에 저장됨

### 두 번째 컴퓨터
1. GitHub Desktop 설치 및 로그인
2. **"File"** → **"Clone Repository"**
3. `jadu329-cmd/Hana_2026` 선택
4. Clone
5. 파일 수정 후 Commit → Push

## 📊 GitHub Desktop vs PowerShell

| 기능 | GitHub Desktop | PowerShell |
|------|---------------|------------|
| 토큰 필요 | ❌ 불필요 | ✅ 필요 |
| 사용 난이도 | ⭐⭐⭐⭐ 매우 쉬움 | ⭐⭐ 보통 |
| GUI | ✅ 있음 | ❌ 없음 |
| 충돌 해결 | ✅ 쉬움 | ⚠️ 어려움 |
| 자동화 | ⚠️ 제한적 | ✅ 스크립트 가능 |
