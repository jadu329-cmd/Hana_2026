# GitHub Personal Access Token이 필요한 이유

## 🔒 왜 비밀번호 대신 토큰을 사용하나요?

### 1. 보안 강화
- **비밀번호는 한 번만 사용**: 비밀번호를 여러 곳에 입력하면 해킹 위험이 증가합니다
- **토큰은 제한적 권한**: 각 토큰은 특정 권한만 가집니다 (예: 저장소 접근만)
- **토큰은 취소 가능**: 토큰이 유출되면 즉시 취소할 수 있습니다
- **비밀번호는 변경 어려움**: 비밀번호를 바꾸면 모든 곳에서 다시 설정해야 합니다

### 2. GitHub의 정책 변경
- **2021년 8월부터**: GitHub는 비밀번호 인증을 완전히 차단했습니다
- **HTTPS 푸시 시**: 비밀번호 대신 Personal Access Token 필수
- **SSH 키 사용 시**: 토큰 불필요 (하지만 SSH 키 설정이 복잡함)

## 🔑 Personal Access Token이란?

### 개념
- GitHub 계정에 접근할 수 있는 **임시 비밀번호** 같은 것
- 특정 권한만 부여할 수 있습니다
- 언제든지 생성/삭제/취소 가능합니다

### 비유
- **비밀번호**: 집 열쇠 (모든 방에 접근 가능)
- **토큰**: 특정 방만 열 수 있는 임시 열쇠

## 📋 토큰 사용 시나리오

### 시나리오 1: 토큰 유출
- 토큰이 유출되면 → 즉시 GitHub에서 취소
- 비밀번호는 그대로 안전
- 다른 서비스에 영향 없음

### 시나리오 2: 프로젝트별 권한
- 프로젝트 A: 읽기만 가능한 토큰
- 프로젝트 B: 읽기/쓰기 가능한 토큰
- 각각 다른 토큰 사용 가능

### 시나리오 3: 토큰 만료
- 30일, 60일, 90일 등 만료 기간 설정 가능
- 만료되면 자동으로 접근 차단
- 보안 자동 관리

## 🆚 비밀번호 vs 토큰 비교

| 항목 | 비밀번호 | Personal Access Token |
|------|---------|---------------------|
| 사용 범위 | 모든 서비스 | GitHub만 |
| 권한 제어 | 불가능 | 가능 (읽기/쓰기 등) |
| 취소 | 비밀번호 변경 필요 | 즉시 취소 가능 |
| 만료 | 없음 | 설정 가능 |
| 보안 | 낮음 (여러 곳 사용) | 높음 (제한적 사용) |

## 💡 실제 사용 예시

### 토큰 생성 시
```
Note: Hana Project
Expiration: 90 days
Scopes: repo (저장소 접근만)
```

### 푸시할 때
```
Username: jadu329-cmd
Password: ghp_xxxxxxxxxxxxxxxxxxxx (토큰)
```

비밀번호가 아니라 토큰을 입력합니다!

## ✅ 결론

**Personal Access Token을 사용하는 이유:**
1. ✅ 보안 강화 (비밀번호 보호)
2. ✅ 권한 제어 (필요한 권한만 부여)
3. ✅ GitHub 정책 (2021년부터 필수)
4. ✅ 관리 용이 (생성/취소 쉬움)

**간단히 말하면:**
- 비밀번호는 집 열쇠 (중요하니까 잘 보관)
- 토큰은 특정 방 열쇠 (잃어버려도 집 열쇠는 안전)
